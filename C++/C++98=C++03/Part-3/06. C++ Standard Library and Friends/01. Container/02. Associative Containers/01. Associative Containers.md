# Associative Containers trong C++ STL

**Associative Containers** (Container liên kết) là một nhóm container hoàn toàn khác với **Sequence Containers** trong C++ STL.

Thay vì lưu trữ dữ liệu theo **thứ tự chèn**, Associative Containers quản lý dữ liệu dựa trên **Key (khóa)**. Mỗi khi thêm phần tử mới, container sẽ **tự động sắp xếp** theo Key.

Bên dưới, hầu hết Associative Containers được triển khai bằng **Red-Black Tree (Cây Đỏ-Đen)**, giúp các thao tác tìm kiếm, thêm và xóa đều có độ phức tạp:

\[
O(\log n)
\]

---

# 1. Phân loại Associative Containers

C++ STL cung cấp bốn Associative Containers chính trong hai thư viện `<set>` và `<map>`.

| Container       | Đặc điểm                                                 |
| --------------- | -------------------------------------------------------- |
| `std::set`      | Lưu các phần tử duy nhất. Giá trị chính là Key.          |
| `std::map`      | Lưu các cặp **Key → Value**. Mỗi Key là duy nhất.        |
| `std::multiset` | Giống `set` nhưng cho phép các giá trị trùng nhau.       |
| `std::multimap` | Giống `map` nhưng một Key có thể ánh xạ đến nhiều Value. |

---

# 2. Các hàm đặc trưng của Associative Containers

Do được xây dựng bằng **Red-Black Tree**, các container này hỗ trợ nhiều hàm tìm kiếm rất mạnh với độ phức tạp:

\[
O(\log n)
\]

---

## 🔍 2.1. Tìm kiếm (Search)

### `find(key)`

Tìm phần tử có Key tương ứng.

**Giá trị trả về**

- Iterator đến phần tử nếu tìm thấy.
- `end()` nếu không tìm thấy.

```cpp
auto it = mySet.find(10);

if (it != mySet.end())
{
    std::cout << "Found";
}
```

---

### `count(key)`

Đếm số phần tử có Key tương ứng.

| Container  | Giá trị trả về |
| ---------- | -------------- |
| `set`      | 0 hoặc 1       |
| `map`      | 0 hoặc 1       |
| `multiset` | ≥ 0            |
| `multimap` | ≥ 0            |

```cpp
std::cout << s.count(5);
```

---

### `contains(key)` (C++20)

Kiểm tra Key có tồn tại hay không.

Trả về:

```cpp
true
```

hoặc

```cpp
false
```

Ví dụ:

```cpp
if (s.contains(20))
{
    std::cout << "Ton tai";
}
```

---

### `lower_bound(key)`

Trả về iterator đến phần tử đầu tiên **không nhỏ hơn** (`>=`) Key.

Ví dụ:

```
1 3 5 7 9
```

```cpp
lower_bound(4)
```

Kết quả:

```
5
```

---

### `upper_bound(key)`

Trả về iterator đến phần tử đầu tiên **lớn hơn** Key.

Ví dụ:

```
1 3 5 7 9
```

```cpp
upper_bound(5)
```

Kết quả:

```
7
```

---

### `equal_range(key)`

Trả về một cặp (`std::pair`) gồm:

- `lower_bound()`
- `upper_bound()`

Thường dùng với:

- `multiset`
- `multimap`

---

# ✏️ 2.2. Chỉnh sửa (Modifiers)

---

### `insert()`

Thêm phần tử.

```cpp
set.insert(10);

map.insert({"Nhan", 9});
```

Container sẽ **tự động** tìm vị trí thích hợp.

---

### `emplace()`

Tạo trực tiếp phần tử trong container.

```cpp
students.emplace("Nhan", 10);
```

Ưu điểm:

- Không tạo bản sao.
- Hiệu năng tốt hơn `insert()`.

---

### `erase()`

Xóa phần tử.

Có thể xóa theo:

- Key

```cpp
set.erase(5);
```

hoặc

- Iterator

```cpp
set.erase(it);
```

---

### `extract()` (C++17)

Tách node ra khỏi container mà **không hủy vùng nhớ**.

Ví dụ:

```cpp
auto node = mySet.extract(5);
```

Sau đó có thể chèn sang container khác.

---

# 🧮 2.3. Truy xuất riêng của `std::map`

---

## `operator[]`

```cpp
map[key]
```

Nếu Key chưa tồn tại:

➡️ Tự động tạo Key mới.

Ví dụ:

```cpp
std::map<std::string,int> score;

score["Nhan"] = 10;
```

Nếu:

```cpp
score["An"];
```

thì map sẽ tự sinh:

```
An -> 0
```

---

## `at()`

```cpp
map.at(key)
```

Nếu Key không tồn tại:

➡️ Ném ngoại lệ

```cpp
std::out_of_range
```

Ví dụ:

```cpp
std::cout << score.at("Nhan");
```

# 3. So sánh Sequence Containers và Associative Containers

| Đặc điểm             | Sequence Containers            | Associative Containers       |
| -------------------- | ------------------------------ | ---------------------------- |
| Ví dụ                | `vector`, `deque`, `list`      | `set`, `map`                 |
| Cơ chế lưu trữ       | Theo thứ tự chèn               | Theo Key                     |
| Thứ tự dữ liệu       | Giữ nguyên thứ tự chèn         | Luôn tự động sắp xếp         |
| Cấu trúc dữ liệu     | Dynamic Array, Linked List     | Red-Black Tree               |
| Tìm kiếm             | `O(n)`                         | `O(log n)`                   |
| Thêm đầu/cuối        | Có (`push_back`, `push_front`) | Không có                     |
| Truy cập bằng chỉ số | Có (`vector`, `deque`)         | Không có                     |
| Duyệt theo thứ tự    | Theo thứ tự chèn               | Theo thứ tự tăng dần của Key |

---

# 4. Độ phức tạp các thao tác

| Thao tác        | Complexity |
| --------------- | ---------- |
| `find()`        | `O(log n)` |
| `insert()`      | `O(log n)` |
| `erase()`       | `O(log n)` |
| `lower_bound()` | `O(log n)` |
| `upper_bound()` | `O(log n)` |
| `contains()`    | `O(log n)` |

# 5. Khi nào nên dùng?

### Dùng `std::set`

Khi:

- Không cần dữ liệu trùng.
- Muốn dữ liệu luôn được sắp xếp.

Ví dụ:

- Danh sách ID.
- Danh sách Email.

---

### Dùng `std::multiset`

Khi:

- Cho phép giá trị trùng.

Ví dụ:

- Điểm thi.
- Giá sản phẩm.

---

### Dùng `std::map`

Khi:

- Cần lưu dạng **Key → Value**.
- Muốn Key luôn được sắp xếp.

Ví dụ:

```
Mã sinh viên → Tên

ID → User

Tên → Điểm
```

---

### Dùng `std::multimap`

Khi:

Một Key có nhiều Value.

Ví dụ:

```
Môn học
    ↓
Danh sách sinh viên
```
