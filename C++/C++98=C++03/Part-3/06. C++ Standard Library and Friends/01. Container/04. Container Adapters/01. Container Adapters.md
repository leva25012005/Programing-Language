# Tổng quan các nhóm Container trong C++ STL

Như đã trình bày ở trên, **Container Adapters** (Bộ thích ứng container) đóng vai trò là một lớp **bao bọc (Wrapper)** bên ngoài các **Sequence Containers**, nhằm giới hạn hoặc thay đổi cách sử dụng của chúng.

Chúng **không phải** là một cấu trúc dữ liệu mới mà chỉ cung cấp một giao diện phù hợp với từng mô hình xử lý dữ liệu như:

- LIFO (Last In First Out)
- FIFO (First In First Out)
- Priority Queue (Hàng đợi ưu tiên)

---

# Bản đồ tổng quan các Container trong C++ STL

```text

                                      │
         ┌────────────────────────────┼────────────────────────────┐

 ┌───────┴───────────┐        ┌───────┴───────────┐        ┌───────┴───────────┐

         ├─ std::vector               ├─ std::set                  ├─ std::unordered_set
         ├─ std::deque                ├─ std::map                  ├─ std::unordered_map
         ├─ std::list                 ├─ std::multiset             ├─ std::unordered_multiset

```

# 1. Sequence Containers

Đây là nhóm container lưu dữ liệu theo **thứ tự tuyến tính**.

## Bao gồm

- `std::vector`
- `std::deque`
- `std::list`
- `std::forward_list`
- `std::array`

## Đặc điểm

- Giữ nguyên thứ tự chèn.
- Có thể truy cập theo iterator.
- Một số hỗ trợ truy cập ngẫu nhiên (`vector`, `deque`, `array`).

### Cấu trúc dữ liệu

- Dynamic Array
- Linked List
- Static Array

### Khi nào sử dụng?

- Danh sách sinh viên.
- Danh sách sản phẩm.
- Mảng dữ liệu.
- Các bài toán xử lý tuần tự.

---

# 2. Associative Containers

Các container trong nhóm này quản lý dữ liệu dựa trên **Key**.

Dữ liệu luôn được **tự động sắp xếp**.

## Bao gồm

- `std::set`
- `std::multiset`
- `std::map`
- `std::multimap`

### Cấu trúc dữ liệu

Red-Black Tree.

### Đặc điểm

- Tìm kiếm:

```
O(log n)
```

- Thêm:

```
O(log n)
```

- Xóa:

```
O(log n)
```

### Khi nào sử dụng?

- Danh bạ.
- Danh sách ID.
- Hệ thống tra cứu.
- Dictionary.

---

# 3. Unordered Containers

Đây là phiên bản sử dụng **Hash Table**.

Không giữ thứ tự.

## Bao gồm

- `std::unordered_set`
- `std::unordered_multiset`
- `std::unordered_map`
- `std::unordered_multimap`

### Cấu trúc dữ liệu

Hash Table.

### Đặc điểm

Tìm kiếm:

```
O(1)
```

(trung bình)

### Khi nào sử dụng?

- Cache.
- Login.
- Password.
- ID.
- Database Index.

---

# 4. Container Adapters

Container Adapter không lưu dữ liệu theo cách riêng.

Chúng sử dụng một Sequence Container phía dưới để thực hiện việc lưu trữ.

---

## `std::stack`

### Mô hình

```
LIFO
```

Last In First Out.

Ví dụ

```
3
2
1
```

Container mặc định:

```
std::deque
```

---

## `std::queue`

### Mô hình

```
FIFO
```

First In First Out.

Ví dụ

```
1
2
3
```

Container mặc định:

```
std::deque
```

---

## `std::priority_queue`

### Mô hình

```
Priority Queue
```

Container mặc định:

```
std::vector
```

Cấu trúc bên dưới:

```
Heap
```

Phần tử có độ ưu tiên cao nhất luôn nằm ở đầu.

---

# So sánh các nhóm Container

| Nhóm        | Cấu trúc dữ liệu            | Có sắp xếp              | Tìm kiếm                | Ví dụ            |
| ----------- | --------------------------- | ----------------------- | ----------------------- | ---------------- |
| Sequence    | Dynamic Array / Linked List | ❌                      | `O(n)`                  | `vector`, `list` |
| Associative | Red-Black Tree              | ✅                      | `O(log n)`              | `map`, `set`     |
| Unordered   | Hash Table                  | ❌                      | `O(1)` (trung bình)     | `unordered_map`  |
| Adapter     | Wrapper                     | Phụ thuộc container gốc | Phụ thuộc container gốc | `stack`, `queue` |
