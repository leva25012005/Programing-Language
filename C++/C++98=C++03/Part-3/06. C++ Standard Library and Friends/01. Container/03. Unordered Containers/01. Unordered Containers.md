# Unordered Containers trong C++ STL

Unordered Containers (Container không thứ tự) **không phải là Sequence Containers**. Tương tự như Associative Containers, chúng thuộc nhóm container quản lý dữ liệu dựa trên **chìa khóa (Key)** chứ không dựa trên vị trí chèn tuyến tính.

Điểm khác biệt cốt lõi là thay vì sắp xếp các phần tử theo thứ tự bằng cấu trúc **Cây (Tree)**, Unordered Containers sử dụng **Bảng băm (Hash Table)**.

Dữ liệu được phân bổ vào các **Bucket** (thùng chứa) dựa trên giá trị của **Hash Code**. Vì vậy, thứ tự hiển thị của các phần tử là hoàn toàn ngẫu nhiên và có thể thay đổi sau khi chèn thêm dữ liệu.

---

# 1. Phân loại Unordered Containers

C++ cung cấp 4 loại Unordered Containers chính (thuộc thư viện `<unordered_set>` và `<unordered_map>`):

| Container                 | Mô tả                                            |
| ------------------------- | ------------------------------------------------ |
| `std::unordered_set`      | Tập hợp các phần tử duy nhất, không theo thứ tự  |
| `std::unordered_map`      | Cặp `Key-Value`, Key duy nhất, không theo thứ tự |
| `std::unordered_multiset` | Cho phép phần tử trùng lặp                       |
| `std::unordered_multimap` | Một Key có thể có nhiều Value                    |

---

# 2. Đặc điểm hoạt động

Khác với `map` hay `set` sử dụng **Red-Black Tree**, nhóm này sử dụng:

> **Hash Table (Bảng băm)**

Quá trình hoạt động:

```
Key
 │
 ▼
std::hash<Key>
 │
 ▼
Hash Code
 │
 ▼
Bucket
 │
 ▼
Lưu dữ liệu
```

Nhờ vậy:

- Tìm kiếm cực nhanh
- Chèn cực nhanh
- Xóa cực nhanh

Trung bình:

```
O(1)
```

Tệ nhất (Hash Collision):

```
O(n)
```

# 3. Các hàm phổ biến

## 3.1. Tìm kiếm

### `find(key)`

Tìm phần tử theo Key.

```cpp
std::unordered_map<int, std::string> mp;

auto it = mp.find(10);

if (it != mp.end())
    std::cout << it->second;
```

---

### `count(key)`

Đếm số phần tử có Key.

```cpp
std::unordered_set<int> s = {1,2,3};

std::cout << s.count(2);
```

Kết quả:

```
1
```

---

### `contains(key)` (C++20)

```cpp
if (mp.contains(10))
{
    std::cout << "Found";
}
```

---

# 4. Chèn dữ liệu

## `insert()`

```cpp
unordered_set<int> s;

s.insert(5);
s.insert(8);
```

---

## `emplace()`

Khởi tạo trực tiếp trong container.

```cpp
unordered_map<int,string> mp;

mp.emplace(1,"Nhan");
```

---

# 5. Xóa dữ liệu

## `erase(key)`

```cpp
mp.erase(10);
```

---

## `erase(iterator)`

```cpp
auto it = mp.find(10);

mp.erase(it);
```

---

# 6. Truy cập (chỉ dành cho unordered_map)

## `operator[]`

Nếu Key chưa tồn tại sẽ **tự động tạo**.

```cpp
unordered_map<string,int> score;

score["Nhan"] = 10;
```

---

## `at()`

Không tạo Key mới.

```cpp
score.at("Nhan");
```

Nếu không tồn tại:

```
std::out_of_range
```

---

# 7. Các hàm quản lý Hash Table

Đây là nhóm hàm chỉ có ở **Unordered Containers**.

---

## `bucket_count()`

Trả về số lượng Bucket.

```cpp
unordered_set<int> s;

std::cout << s.bucket_count();
```

---

## `bucket_size()`

Cho biết Bucket thứ n chứa bao nhiêu phần tử.

```cpp
s.bucket_size(3);
```

---

## `bucket(key)`

Cho biết Key đang nằm ở Bucket nào.

```cpp
std::cout << s.bucket(10);
```

---

## `load_factor()`

Trả về:

```
Số phần tử
------------
Số Bucket
```

Ví dụ:

```
100 phần tử

20 bucket

Load Factor = 5
```

Load Factor càng lớn:

- Collision càng nhiều
- Hiệu năng càng giảm

---

## `max_load_factor()`

Lấy hoặc thay đổi Load Factor tối đa.

```cpp
mp.max_load_factor(0.75);
```

---

## `rehash(n)`

Tạo lại Hash Table.

```cpp
mp.rehash(200);
```

Ý nghĩa:

- Tăng Bucket
- Phân bố lại toàn bộ dữ liệu

---

## `reserve(n)`

Đặt trước dung lượng.

```cpp
unordered_map<int,int> mp;

mp.reserve(100000);
```

Lợi ích:

- Giảm số lần Rehash
- Tăng tốc khi thêm nhiều dữ liệu

---

# 8. Hash Collision

Hash Collision xảy ra khi:

```
Key A
      \
       > cùng Hash Code
      /
Key B
```

Ví dụ:

```
Hash(10)

↓

Bucket 5

Hash(20)

↓

Bucket 5
```

Hai phần tử cùng nằm trong một Bucket.

Khi Collision nhiều:

```
O(1)

↓

O(n)
```

# 9. Khi nào nên dùng?

Dùng khi:

- Không cần sắp xếp dữ liệu
- Muốn tốc độ tìm kiếm nhanh nhất
- Dữ liệu lớn
- Tra cứu theo Key

Ví dụ:

- Username
- Password
- Student ID
- Product ID
- Dictionary
- Cache

---

# 10. Khi nào không nên dùng?

Không phù hợp nếu:

- Muốn duyệt theo thứ tự tăng dần
- Cần `lower_bound()`
- Cần `upper_bound()`
- Cần tìm phần tử gần nhất

Lúc đó nên dùng:

- `std::map`
- `std::set`

---

# 11. So sánh Associative vs Unordered

| Đặc điểm               | Associative (`map`, `set`) | Unordered (`unordered_map`, `unordered_set`) |
| ---------------------- | -------------------------- | -------------------------------------------- |
| Cấu trúc               | Red-Black Tree             | Hash Table                                   |
| Thứ tự                 | Có sắp xếp                 | Không sắp xếp                                |
| Độ phức tạp trung bình | O(log n)                   | O(1)                                         |
| Trường hợp xấu nhất    | O(log n)                   | O(n)                                         |
| Key yêu cầu            | Toán tử `<`                | `std::hash` và `==`                          |
| `lower_bound()`        | Có                         | Không                                        |
| `upper_bound()`        | Có                         | Không                                        |
| `contains()`           | Có (C++20)                 | Có (C++20)                                   |
| Iterator               | Theo thứ tự tăng dần       | Thứ tự ngẫu nhiên                            |

---

# 12. Ví dụ hoàn chỉnh

```cpp
#include <iostream>
#include <unordered_map>

using namespace std;

int main()
{
    unordered_map<string,int> score;

    score["Nhan"] = 9;
    score["An"] = 10;
    score["Binh"] = 8;

    cout << "Danh sach:\n";

    for (auto &x : score)
    {
        cout << x.first << " : " << x.second << endl;
    }

    cout << "\nTim An:\n";

    auto it = score.find("An");

    if (it != score.end())
    {
        cout << it->first << " " << it->second << endl;
    }

    cout << "\nBucket Count: " << score.bucket_count() << endl;
    cout << "Load Factor : " << score.load_factor() << endl;

    return 0;
}
```
